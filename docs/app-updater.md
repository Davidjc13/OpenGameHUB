# App updater

## Goal

Let installations published on [GitHub Releases](https://github.com/Davidjc13/OpenGameHUB/releases) update without manually downloading from the site.

## Component

File: `Services/AppUpdateService.cs`

Model: `AppReleaseInfo` (tag, HTML URL, download URL, asset name, size).

## Current version

`AppUpdateService.CurrentVersion` reads:

1. `AssemblyInformationalVersionAttribute` (without `+commit` suffix)
2. Fallback: three-component `AssemblyVersion`
3. Development: `dev` by default in `.csproj`

Release builds embed the version via `build-installer.ps1`:

```powershell
-p:InformationalVersion=alpha-0.0.7
-p:Version=0.0.7
```

Only **InformationalVersion** carries the `alpha-` prefix; `AssemblyVersion` uses numeric semver for .NET compatibility.

## Check for updates

### API

```
GET https://api.github.com/repos/Davidjc13/OpenGameHUB/releases?per_page=1
```

**Why not `/releases/latest`:** current releases are `prerelease: true` (`alpha-*`). GitHub's `latest` endpoint **ignores** prereleases and would return 404.

The first non-`draft` release from the list is used (API chronological order).

### Expected asset

Name: `OpenGameHUB-Setup-*.exe` (generated by Inno Setup in CI).

### Version comparison

`ReleaseVersionComparer` in the same file:

- Parses `alpha-`, `beta-`, or stable prefixes
- Compares numeric `System.Version`
- If channels differ, orders alpha < beta < stable

`dev` build: `IsNewer` is always true if a release exists (allows testing in Settings).

## UI flows

### On startup (`MainWindowViewModel`)

```
CheckForAppUpdateOnStartupAsync
  if IsDevBuild → exit
  if newer release → StatusText with AppUpdateAvailableHint (12 s)
```

### In Settings (`SettingsViewModel`)

| Action | Behavior |
|--------|----------|
| On open | Automatic `CheckForUpdatesAsync` |
| Check for updates | Same manual check |
| Download and install | `DownloadInstallerAsync` + `LaunchInstallerAndExit` |

Progress bar: indeterminate when querying API; percentage during download.

## Download

- Destination: `%TEMP%\OpenGameHUB\updates\{assetName}`
- If the file exists, tries to delete it; on failure, uses a GUID-suffixed name
- `HttpClient` with User-Agent `OpenGameHUB-Updater/1.0`, 10 min timeout

## Installation

```csharp
Process.Start(installer, "/SILENT /CLOSEAPPLICATIONS /RESTARTAPPLICATIONS");
Environment.Exit(0);
```

Inno Setup (`installer/OpenGameHUB.iss`):

- Same `AppId` across versions → in-place upgrade
- `PrivilegesRequired=lowest` — no admin
- Default directory: `%LocalAppData%\Programs\OpenGameHUB`

**Why exit the app:** the installer must replace the running `OpenGameHUB.exe`; `CLOSEAPPLICATIONS` closes blocking processes.

## CI

`.github/workflows/build-installer.yml` — tag push → `build-installer.ps1 -AppVersion $tag` → attaches exe to the Release.

The updater and CI assume the **same asset name** and repository `Davidjc13/OpenGameHUB`.

## Known limitations

- Windows only (`.exe` installer)
- Requires internet access to `api.github.com` + `github.com`
- Does not verify digital signature of the installer (trusts HTTPS from official GitHub)
- Does not update legendary or user data — only the OpenGameHUB binary
