# Platform integrations

How OpenGameHUB obtains data from each store.

Legend:

- **Official API** — public (or semi-documented) publisher endpoint
- **Local scan** — files, registry, or executables on the PC
- **Cache / logs** — data the launcher already stored locally (sometimes encrypted)
- **Third-party tool** — legendary (Epic), GameLib/GameFinder

## Summary

| Platform | Installed | Cloud library | Install from app | Playtime | Covers |
|----------|-----------|---------------|------------------|----------|--------|
| Steam | GameLib + local VDF | Web API or VDF | `steam://install` | Web API | Steam CDN |
| Epic | GameLib + `.item` manifests | legendary `list --json` | Epic Launcher protocol | — | Metadata pipeline |
| GOG | GameLib | — | GameLib | — | Metadata pipeline |
| Ubisoft | GameLib | Protobuf/YAML cache | `uplay://install` | — | Ubisoft CDN |
| EA | GameLib + registry/XML | Encrypted cache + logs | `link2ea://` / EA Desktop | — | Metadata pipeline |
| Battle.net | GameLib | — | GameLib | — | Metadata pipeline |
| Rockstar | GameLib | — | GameLib | — | Metadata pipeline |
| Riot | GameLib | — | GameLib | — | Static Riot map |
| Xbox / Game Pass | GameFinder Appx | — | exe / shell URI | — | Metadata pipeline |

## Steam

### Installed

- **GameLib** `LauncherManager` — standard Steam library
- Files: `Services/GameLibraryService.cs` → `ScanInstalledGames`

### Cloud library (not installed)

**Option A — official API (recommended for playtime):**

- `SteamWebApiService.GetOwnedGamesAsync`
- Requires API key + SteamID64 in Settings
- Endpoint: `api.steampowered.com`

**Option B — without API:**

- `SteamLocalAccountReader` — active account in `loginusers.vdf`
- `SteamLocalLibraryReader` — AppIds in `localconfig.vdf` and `librarycache/*.json`
- `SteamStoreClient` — names from the Steam store

**Why both:** many users do not want to create an API key; local reading lists owned games but with less metadata and no reliable remote playtime.

### Installation

- `steam://install/{appId}` via `SteamCloudLibraryProvider` / `LaunchSpec.Protocol`

### Playtime

- Only with Web API configured (`EnrichPlaytimeFromOwned`)

## Epic Games

See dedicated document: [epic-and-legendary.md](epic-and-legendary.md).

Summary:

- Installed: GameLib + `EpicManifestScanner` (`C:\ProgramData\Epic\...\Manifests\*.item`)
- Cloud: legendary
- Install: `com.epicgames.launcher://` protocol (does not wait for download)

## Ubisoft Connect

### Installed

- GameLib

### Cloud library

- **Local cache** at `%LocalAppData%\Ubisoft Game Launcher\cache\configuration\configurations`
- `UbisoftCatalogReader` — protobuf + YAML (`UbisoftCatalogModels.cs`)
- No Ubisoft online API in OpenGameHUB

### Installation

- `uplay://install/{uplayId}`

### Covers

- URLs in cache (`ubistatic3-a.akamaihd.net`) → `CatalogCoverUrl` → optional download

---

## EA / Origin / EA App

See dedicated document: [ea-desktop.md](ea-desktop.md).

Summary:

- Installed: GameLib + `EaDesktopScanner` (registry + `installerdata.xml`)
- Cloud: encrypted `InstallInfo` cache (GameFinder decrypt) + log fallback
- Install: `origin2://` / `link2ea://` via EA Desktop
- Onboarding: `EaLibraryPromptWindow` when cache is missing or undecryptable

---

## GOG, Battle.net, Rockstar, Riot

- **Installed only** via GameLib (no cloud provider)
- Launch per `LaunchSpec` generated by GameLib
- Riot: predefined covers in `RiotCoverClient`

---

## Xbox PC / Game Pass

### Installed only

- `XboxGamePassScanner` + **GameFinder.StoreHandlers.Xbox**
- Reads Appx manifests in `WindowsApps`, uses PowerShell `Get-AppxPackage` in some cases
- Launch: executable, `shell:AppsFolder\...`, or `XboxPcApp` with ApplicationId

**Why no cloud:** Game Pass catalog is not exposed in a stable way for third-party apps on Windows like the Steam API.

## Legal transparency

OpenGameHUB is **not affiliated** with any of these platforms. It uses:

- Public APIs where the user provides credentials (Steam, IGDB, SteamGridDB)
- Data already on the user's machine (caches, logs, registry)
- Community tools (legendary) under the user's responsibility

Official launchers are still required to install and play.
